# Contributing to VECTRON

## Getting Started
1. Fork the repository.
2. Clone your fork locally.
3. Install dependencies inside `vectron-app/client` and `vectron-app/server`.
4. Start the app locally and confirm both frontend and backend are running.

## Development Workflow
1. Create a feature branch from `main`.
2. Keep changes focused on one feature or fix per branch.
3. Run local builds before opening a pull request.
4. Update docs when endpoints, MCP tools, or user workflows change.
5. Include screenshots or screen recordings for UI changes when possible.

## Pull Request Checklist
- Code builds successfully
- Relevant docs updated
- No unrelated files included
- Clear PR title and summary
- Screenshots added for UI updates

## Reporting Issues
Use the issue templates in `.github/ISSUE_TEMPLATE/` for:
- Bug reports
- Feature requests

## Code Style
- Prefer TypeScript-first changes
- Keep components modular and focused
- Reuse shared helpers instead of duplicating logic
- Preserve existing visual language unless intentionally redesigning
